앱 페이지 관리

전체 홈페이지 타입과 마법사 구조에 대한 기획 문서입니다. 현재는 기능 구현이 아닌 운영 기준 문서로 제공합니다.

개요

이 방향은 AURA를 단순 페이지가 아니라 “페이지 타입/템플릿 기반 앱 빌더”로 확장하는 구조입니다. 현재 관리자 화면이 문구, API key, 사진, 스타일로 나뉘고 있으므로, 그 위에 앱 페이지 관리를 최상위 개념으로 두는 것이 자연스럽습니다.

관리 섹션 구성

앱 페이지 관리
- 현재 적용 타입: A1
- 타입 목록: A1, A2, B1, B2, B3
- 새 타입 생성
- 기존 타입 복제
- 타입 이름/설명 변경
- 타입 삭제
- 타입 적용
- 미리보기

페이지 타입의 의미

각 타입은 하나의 “홈페이지 설정 묶음”입니다.

A1 타입
- 문구 설정
- 사진 설정
- 스타일 설정
- 채팅/전문가 버튼 설정
- 섹션 구성
- 기본 언어
- 첫 화면 자동 전환 여부
- API 연결 방식

관리 흐름

  1. 앱 페이지 관리에서 A1 선택
  2. 문구관리로 가면 A1의 문구를 편집
  3. 사진관리로 가면 A1의 사진을 편집
  4. 스타일관리로 가면 A1의 스타일을 편집
  5. A2 선택 시 모든 관리 화면이 A2 기준으로 전환

타입 예시

  • A1: 메이크업 중심 앱
  • A2: 헤어/메이크업 혼합 앱
  • B1: 패션 에디토리얼 앱
  • B2: 인테리어 큐레이션 앱
  • B3: 전문가 상담 랜딩형 앱

구현 시 주의점

  1. 삭제는 바로 삭제하지 말고 비활성/보관 처리 추천
  2. A1 같은 기본 타입은 삭제 불가로 잠금 처리 추천
  3. 새 타입 생성보다 “A1 복제 후 A2로 수정” 흐름이 중요
  4. 편집 중인 타입과 실제 적용 타입은 분리 추천

API key 관리

AI 상담용 LLM을 복수 선택하고, provider별 API key와 LLM 버전을 관리합니다. 저장된 key는 전체 노출하지 않고 마스킹합니다.

추후 질문유형 관리 DB 설계

질문유형은 언어별로 분리해 관리할 예정입니다. LLM provider 설정과 독립된 구조로 두면, 같은 질문유형을 ChatGPT/Gemini/Claude 중 선택된 모델에 공통 적용할 수 있습니다.

questionTypes

  - id: makeup_consult
  - label: 메이크업 상담
  - promptType: consultation
  - enabled: true
- en
  - id: makeup_consult
  - label: Makeup consultation
  - promptType: consultation
  - enabled: true
